# 开发说明

这是一份给维护者看的当前开发入口，重点回答三件事：做了什么、为什么这么做、做了以后会怎么样。

## 本轮更新

- 站点统计的字数改为直接统计 `user/posts/` 下所有 Markdown 文章的正文，不再依赖前端伪字段，所以新增文章后字数会真实增长。
- 深色模式下的文章正文、关于页和 Markdown 预览统一改为 CSS 变量驱动，避免出现偏深蓝、偏浅蓝的难读文本。
- 页面布局加宽，左右侧栏不再内部滚动，详情页内容区也更宽，整体更接近 Firefly 的层级感，但保留了一点更明显的个性化布局。
- 顶部导航栏的搜索框改为常驻显示，图标位于输入框内部，减少操作步骤。
- 文章详情页继续显示大纲，归档/分类/标签页继续显示日历，让不同页面承担不同的信息结构。

## 为什么这么做

- 让 Firefly 风格真正落地到阅读体验，而不是只停留在卡片外观。
- 提高长文阅读舒适度，尤其是在深色模式下。
- 让统计、搜索、侧栏和页面切换的行为更稳定、更可预测。
- 在统一感和个性化之间保留空间，而不是把所有页面做成同一种模板。

## 这样做之后会怎样

- 你会看到统计数字会随文章内容变化，不再卡在 0。
- 文章页在暗色模式下更容易读，视觉噪音更少。
- 页面会更宽、更像一个完整博客，左右卡片不再像贴在边上的小面板。
- 搜索、分类、文章大纲和日历的职责会更清楚，信息密度更合理。
